# SFJM Workspace — Product Vision

## Product

SFJM Workspace

SFJM means `Stateful Flow & Journey Management`.

Its internal origin is `Stop Fucking Junior Mode`.

## Purpose

The SFJM Workspace is an operational interface for cognitive and operational continuity between AI conversations.

It must help the user answer quickly:

> Posso continuar com segurança agora?

The product is not a landing page, sales page or generic dashboard. It is an operational environment organized around continuity state, the next safe action and preservation of the journey.

## Core value

The Workspace should reduce regression between conversations by preserving and exposing:

- current state;
- relevant context;
- decisions already made;
- evidence;
- governance;
- canonical sources;
- handoffs;
- the single next safe action.

## Primary product hierarchy

1. Estado de Continuidade
2. Próxima Ação Segura
3. Contextos Preservados
4. Visão da Jornada
5. Fontes Canônicas
6. Evidências, handoffs, auditorias e governança
7. Projeto associado

The continuity state must be more prominent than the project name.

## Home question

> De onde você precisa continuar hoje?

The user should not need to search manually for where work stopped.

## Initial product scope

The first product version may include:

- Home / Continue;
- Estado de Continuidade;
- Próxima Ação Segura;
- Contextos Preservados;
- Visão da Jornada;
- Projetos;
- Handoffs;
- Evidências;
- Auditorias;
- Governança;
- Fontes Canônicas;
- Linha do Tempo;
- Configurações.

## Explicit non-goals for the current stage

- production architecture;
- authentication;
- database persistence;
- automatic synchronization;
- real external integrations;
- protocol modification;
- invented confidence, health or continuity claims.
